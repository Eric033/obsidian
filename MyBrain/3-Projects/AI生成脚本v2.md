# 接口自动化测试生成系统 — 详细实现说明

> 本文档面向 Claude Code，描述一个基于流量日志 + 源码分析的接口自动化测试脚本生成系统的完整实现方案。

---

## 一、系统概述

### 1.1 目标

开发一个命令行工具，自动化生成接口自动化脚本
输入以下三类信息：

1. **流量日志**（抓包 / HAR / 测试框架日志）
2. **被测系统源代码**（Java / Python / Go）
3. **数据库 Schema**（可选，增强断言和数据准备）

自动输出：

- 可直接执行的接口测试脚本（pytest 格式）
- 包含 Setup / Teardown / 断言 / 接口串联的完整用例

### 1.2 技术栈

- **语言**：Python 3.11+
- **测试框架**：pytest
- **AST 解析**：javalang（Java）、Python ast 模块、tree-sitter（多语言通用）
- **项目结构**：模块化，每个分析引擎独立为一个模块
- 开发过程全程使用uv 虚拟环境
- 基于文件数据库
- 单元测试具备
